# ТЗ: Android приложение для отправки времени на ESP32 часы

**Статус:** В разработку  
**Приоритет:** Высокий  
**Платформа:** Android (минимум API 26, рекомендуется API 31+)  
**Язык:** Kotlin или Java  

---

## 📌 Цель

Разработать Android приложение, которое:
1. Сканирует BLE устройства в поиске **"esp32-clock"**
2. Подключается к найденному устройству
3. Отправляет текущее системное время
4. Отключается после успешной отправки

**Результат:** Часы на ESP32 синхронизируют время с телефоном автоматически.

---

## 🔧 Технические требования

### BLE Параметры (НЕИЗМЕНЯЕМЫЕ)

| Параметр | Значение |
|----------|----------|
| **Device Name** | `esp32-clock` |
| **Service UUID** | `4fafc201-1fb5-459e-8fcc-c5c9c331914b` |
| **Characteristic UUID** | `beb5483e-36e1-4688-b7f5-ea07361b26a8` |
| **Characteristic Properties** | READ, WRITE |
| **Data Size** | ровно 8 байт |
| **Data Format** | Big-Endian (MSB first) |
| **Data Type** | Unix время в миллисекундах (uint64_t) |

### Функциональные требования

#### 1. Сканирование BLE устройств
- **Что делать:** Инициировать сканирование BLE устройств
- **Критерий:** Найти и подключиться к устройству с именем `esp32-clock`
- **Таймаут:** Макс. 10 секунд сканирования, затем отмена
- **Результат:** Вывести статус поиска в UI (ищем... / найдено / ошибка)

#### 2. Подключение к устройству
- **Что делать:** Установить GATT соединение с найденным устройством
- **Callback:** `BluetoothGattCallback`
- **Логирование:** Логировать события подключения/отключения
- **Обработка ошибок:** Повторить попытку 1-2 раза при ошибке

#### 3. Обнаружение сервисов
- **Что делать:** После подключения вызвать `gatt.discoverServices()`
- **Ожидание:** Дождаться `onServicesDiscovered()`
- **Поиск:** Найти Service с UUID `4fafc201-1fb5-459e-8fcc-c5c9c331914b`
- **Проверка:** Убедиться, что сервис найден, иначе — отключиться

#### 4. Поиск характеристики
- **Что делать:** В найденном сервисе найти Characteristic с UUID `beb5483e-36e1-4688-b7f5-ea07361b26a8`
- **Проверка:** Убедиться, что свойства включают WRITE
- **Обработка ошибок:** Если не найдена — отключиться и сообщить об ошибке

#### 5. Конвертация и отправка времени
- **Получить время:** `System.currentTimeMillis()` → получить миллисекунды
- **Конвертировать в 8 байт** (Big-Endian):
  ```
  timeBytes[0] = (byte) ((unixMillis >> 56) & 0xFF)
  timeBytes[1] = (byte) ((unixMillis >> 48) & 0xFF)
  timeBytes[2] = (byte) ((unixMillis >> 40) & 0xFF)
  timeBytes[3] = (byte) ((unixMillis >> 32) & 0xFF)
  timeBytes[4] = (byte) ((unixMillis >> 24) & 0xFF)
  timeBytes[5] = (byte) ((unixMillis >> 16) & 0xFF)
  timeBytes[6] = (byte) ((unixMillis >> 8) & 0xFF)
  timeBytes[7] = (byte) (unixMillis & 0xFF)
  ```
- **Отправить:** `characteristic.value = timeBytes` → `gatt.writeCharacteristic(characteristic)`
- **Callback:** Обработать результат в `onCharacteristicWrite()`

#### 6. Завершение и отключение
- **После отправки:** Вызвать `gatt.disconnect()`
- **После отключения:** Закрыть соединение `gatt.close()`
- **Статус:** Вывести сообщение об успехе в UI

#### 7. Обработка ошибок
- **Сканирование не найдено:** "Устройство не найдено за 10 сек"
- **Ошибка подключения:** "Ошибка подключения: [код ошибки]"
- **Ошибка обнаружения сервисов:** "Сервис не найден"
- **Ошибка отправки:** "Ошибка отправки времени"
- **Таймаут операции:** Добавить общий таймаут 30 сек (отмена операции)

## 📋 Требования к разрешениям

Добавить в `AndroidManifest.xml`:

```xml
<uses-permission android:name="android.permission.BLUETOOTH" />
<uses-permission android:name="android.permission.BLUETOOTH_ADMIN" />
<uses-permission android:name="android.permission.BLUETOOTH_SCAN" />
<uses-permission android:name="android.permission.BLUETOOTH_CONNECT" />
<uses-feature android:name="android.hardware.bluetooth_le" android:required="true" />
```

**Runtime Permissions (API 31+):**
- Запросить `BLUETOOTH_SCAN` и `BLUETOOTH_CONNECT` при первом запуске
- Проверять `ContextCompat.checkSelfPermission()` перед каждой операцией

### BleManager.kt
```kotlin
class BleManager(private val context: Context) {
    fun startScan(onResult: (success: Boolean, device: BluetoothDevice?) -> Unit)
    fun connect(device: BluetoothDevice)
    fun discoverServices()
    fun sendTime()
    fun disconnect()
}
```

### Обязательные методы в callback:
- `onConnectionStateChange()` → подключение/отключение
- `onServicesDiscovered()` → обнаружение сервисов
- `onCharacteristicWrite()` → результат отправки

## ⚠️ Важные замечания

1. **КРИТИЧНО:** Размер данных ровно **8 байт**, иначе ESP32 проигнорирует
2. **КРИТИЧНО:** Порядок байт — **Big-Endian** (от старшего к младшему)
3. **КРИТИЧНО:** Время в **миллисекундах**, а не в секундах
4. **На всякий случай:** После успешной отправки обязательно отключиться
5. **Для отладки:** Используйте nRF Connect для тестирования BLE
6. **Для отладки:** Проверьте разрешения перед операциями BLE
7. **Для отладки:** Используйте `adb logcat` для просмотра логов
